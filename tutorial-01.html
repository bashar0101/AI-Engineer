Professor's Opening

As a Computer Engineer, you have seen matrices in circuit analysis and signal processing. Now we're going to re-examine linear algebra through the lens of AI — where a vector is not just a mathematical object, but a representation of meaning, and a matrix is a learned transformation. This shift in perspective is everything.

Section 1

Scalars, Vectors & High-Dimensional Spaces

In AI, a scalar is any single number — a loss value, a learning rate, a confidence score. A vector is an ordered list of numbers, but more importantly, it is a point in space — or a direction and magnitude.

What is a Vector in AI Context?

When a language model processes the word "king", it doesn't see letters — it sees a vector of 768 or 1024 floating-point numbers. This vector encodes the word's meaning in high-dimensional space. Words with similar meanings have vectors that point in similar directions.

🤖 AI Connection

Word embeddings (Word2Vec, GloVe, BERT embeddings) are vectors. Image features from a CNN are vectors. A user's preferences in a recommendation system are vectors. Everything in ML eventually becomes a vector.

Vector Notation
x = [x₁, x₂, x₃, ..., xₙ] ← a vector in ℝⁿ

Example — word embedding of "king":
king = [0.25, -0.13, 0.87, 0.04, ..., -0.32] ← 768 dimensions

// "queen" would be a vector very close to "king" in this 768D space

Vector Operations

The operations you'll use daily in ML:

Essential Vector Operations
// Addition: combine two information sources
a + b = [a₁+b₁, a₂+b₂, ..., aₙ+bₙ]

// Scalar multiplication: scale an embedding
α · v = [α·v₁, α·v₂, ..., α·vₙ]

// Magnitude (L2 norm): how "big" is this vector?
‖v‖ = √(v₁² + v₂² + ... + vₙ²)
Python — NumPy 
import numpy as np

# Define two vectors (imagine word embeddings)
king  = np.array([0.25, -0.13, 0.87, 0.04])
queen = np.array([0.22, -0.10, 0.89, 0.06])
man   = np.array([0.60,  0.45, 0.11, 0.80])
woman = np.array([0.58,  0.43, 0.13, 0.78])

# The famous analogy: king - man + woman ≈ queen
result = king - man + woman
print(result)  # should be close to queen

# Vector magnitude (L2 norm)
magnitude = np.linalg.norm(king)
print(f"Magnitude of 'king' vector: {magnitude:.3f}")

# Normalized vector (unit vector — direction only)
king_unit = king / np.linalg.norm(king)
print(f"Unit vector norm: {np.linalg.norm(king_unit):.3f}")  # always 1.0
💡

CE to AI Mindset Shift

In signal processing you worked with 1D, 2D, or 3D vectors. In AI, vectors routinely have 768, 1024, or 4096 dimensions. You can't visualize them, but the math is identical. Your brain doesn't need to picture it — the operations are the same.

Section 2

Matrices & Their Operations

A matrix is a 2D array of numbers — but in AI, it's almost always a transformation or a set of learned parameters. The weight matrix in a neural network layer is literally a matrix that transforms one vector space into another.

Matrix Anatomy

An m × n matrix has m rows and n columns. In deep learning, a weight matrix W of shape (input_dim × output_dim) transforms an input vector into an output vector.

3×3 Matrix A

[ 123 ]
456
[ 789 ]

Matrix Multiplication — The Core Operation

Matrix multiplication is the fundamental operation in deep learning. Every forward pass through a neural network layer is essentially: output = activation(W · input + b)

Matrix Multiplication Rule
C = A · B where C[i,j] = Σₖ A[i,k] · B[k,j]

Shape rule: (m×k) · (k×n) = (m×n)

// Inner dimensions must match! This is why layer shapes matter so much in PyTorch.
Python — NumPy 
import numpy as np

# Simulating one neural network layer forward pass
batch_size   = 32    # 32 samples
input_dim    = 512   # input features per sample
output_dim   = 256   # neurons in this layer

# Input: 32 samples, each with 512 features → shape (32, 512)
X = np.random.randn(batch_size, input_dim)

# Weight matrix (learned params) → shape (512, 256)
W = np.random.randn(input_dim, output_dim) * 0.01

# Bias → shape (256,)
b = np.zeros(output_dim)

# Forward pass: (32,512) · (512,256) + (256,) = (32,256)
output = X @ W + b   # @ is matrix multiply in NumPy
print(f"Output shape: {output.shape}")  # (32, 256) ✓

# Transpose — used in backpropagation
W_T = W.T  # shape (256, 512)
print(f"Transposed W: {W_T.shape}")
🔧 CE Analogy: Matrix as Circuit Transform

Think of a weight matrix like a transfer function in your circuits course. It takes a signal (input vector) and transforms it into another signal (output vector). Neural network training is finding the best "transfer function" (weight matrix) that maps inputs to the desired outputs. Backprop is just gradient-based tuning of this function.

Section 3

Dot Product & Cosine Similarity

The dot product is arguably the most used operation in all of AI. It measures how aligned two vectors are — and alignment is a proxy for similarity.

Dot Product & Cosine Similarity
a · b = Σᵢ aᵢ · bᵢ = ‖a‖ · ‖b‖ · cos(θ)

Cosine similarity = (a · b) / (‖a‖ · ‖b‖)

// Result is between -1 and 1:
// +1 → same direction (very similar)
// 0 → perpendicular (unrelated)
// -1 → opposite direction (opposites)
🤖 AI Connection — Attention Mechanism

The Transformer's attention mechanism is fundamentally a set of dot products. When processing the word "bank" in "I went to the river bank", the model computes dot products between the query vector of "bank" and the key vectors of every other word — finding which words are most relevant (highest dot product = most attended to). This is how GPT-4 and Claude work at the core.

Python — Semantic Search using Cosine Similarity 
import numpy as np

def cosine_similarity(a, b):
    """Measure how similar two vectors are."""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Simulated word embeddings (in reality these are 768-dim)
dog   = np.array([0.9, 0.1, 0.05, 0.8])   # "animal-like"
cat   = np.array([0.85, 0.12, 0.07, 0.75]) # similar to dog
car   = np.array([0.1, 0.9, 0.95, 0.05])   # very different

print(f"dog vs cat: {cosine_similarity(dog, cat):.3f}")   # high ~0.99
print(f"dog vs car: {cosine_similarity(dog, car):.3f}")   # low ~0.15

# This is how semantic search works!
# Query: "I want a pet" → find top-k most similar vectors in database
query = np.array([0.88, 0.11, 0.06, 0.77])
database = {"dog": dog, "cat": cat, "car": car}
scores = {word: cosine_similarity(query, vec) for word, vec in database.items()}
best = max(scores, key=scores.get)
print(f"Most relevant to query: '{best}' (score: {scores[best]:.3f})")
Section 4

Linear Transformations

A matrix A applied to a vector x performs a linear transformation — it rotates, scales, shears, or projects the vector in space. Neural network layers are linear transformations (followed by non-linear activation functions).

Identity, Inverse, and Determinant

Identity Matrix (I)
Leaves every vector unchanged. Like multiplying by 1. Used as a baseline and in residual connections (ResNets).
Inverse Matrix (A⁻¹)
Undoes the transformation of A. Not all matrices are invertible. Critical in optimization math.
Determinant (det(A))
Measures how much the transformation scales volume. det = 0 means the matrix collapses space (rank-deficient — bad for training).
Transpose (Aᵀ)
Flips rows and columns. Essential in backpropagation — gradient flows backwards through the transposed weight matrix.
🤖 AI Connection — Residual Connections

ResNet (the architecture behind many modern vision models) adds the input back to the output of a layer: output = F(x) + x. The "+ x" part is adding the identity transformation — it lets gradients flow unchanged through many layers, solving the vanishing gradient problem. This is pure linear algebra intuition.

Section 5

Eigenvalues & Eigenvectors

An eigenvector of matrix A is a special vector that, when transformed by A, only scales — it doesn't rotate. The scale factor is the eigenvalue.

Eigendecomposition
A · v = λ · v

where:
· v = eigenvector (direction preserved under A)
· λ (lambda) = eigenvalue (how much v is stretched/shrunk)

// A matrix can be decomposed: A = Q · Λ · Q⁻¹
// Q = matrix of eigenvectors, Λ = diagonal matrix of eigenvalues
🤖 AI Connection — PCA & Covariance

Principal Component Analysis (PCA) — one of the most important dimensionality reduction techniques — is entirely about computing eigenvectors of the covariance matrix. The eigenvectors are the "principal components" (directions of maximum variance), and the eigenvalues tell you how much information each direction captures. Used for: data visualization, noise reduction, feature compression, and more.

Python — Eigendecomposition with NumPy 
import numpy as np

# A covariance matrix (symmetric positive semi-definite)
A = np.array([[4, 2],
              [2, 3]])

eigenvalues, eigenvectors = np.linalg.eig(A)
print("Eigenvalues:", eigenvalues)    # [5.56, 1.44]
print("Eigenvectors:\n", eigenvectors) # columns are eigenvectors

# Verify: A·v = λ·v
v0 = eigenvectors[:, 0]  # first eigenvector
l0 = eigenvalues[0]      # first eigenvalue
print("A·v0:", A @ v0)
print("λ·v0:", l0 * v0)     # should be the same!

# PCA on random data (dimensionality reduction 4D → 2D)
from sklearn.decomposition import PCA
data = np.random.randn(100, 4)     # 100 samples, 4 features
pca = PCA(n_components=2)
reduced = pca.fit_transform(data)  # 100 samples, 2 features
print(f"Variance retained: {pca.explained_variance_ratio_.sum():.1%}")
Section 6

Singular Value Decomposition (SVD)

SVD is the generalization of eigendecomposition to non-square matrices. It's one of the most powerful tools in numerical linear algebra and shows up everywhere in modern AI.

SVD Decomposition
A = U · Σ · Vᵀ

where (for an m×n matrix A):
· U = m×m orthogonal matrix (left singular vectors)
· Σ = m×n diagonal matrix (singular values, always ≥ 0)
· Vᵀ = n×n orthogonal matrix (right singular vectors)

// Truncated SVD: keep only top-k singular values → compressed representation
🤖 AI Connection — LoRA Fine-tuning

LoRA (Low-Rank Adaptation) — the dominant method for fine-tuning large language models cheaply — uses SVD intuition. Instead of modifying the full weight matrix W (billions of parameters), it approximates the change ΔW as a product of two small matrices: ΔW = A · B. This is a low-rank approximation, exactly like truncated SVD. Fine-tuning GPT-4-level models on a single GPU with LoRA is possible because of this math.

⭐

Summary: Why Linear Algebra is the Backbone of AI

Neural networks are compositions of linear transformations (matrix multiplications) and non-linearities (activation functions). Embeddings are vectors. Attention is dot products. PCA is eigenvectors. LoRA is SVD. Virtually every major AI technique traces directly back to the concepts in this tutorial. Master these, and you understand what's happening under the hood of every model you'll ever use.

Broadcasting
NumPy automatically stretches smaller arrays to match shapes. Critical for efficient batch operations in deep learning.
Batch Matrix Multiply
Processing multiple matrices simultaneously. In PyTorch: torch.bmm(). Every training step processes a "batch" this way.
Hadamard Product
Element-wise multiplication (not matrix multiply). Used in LSTM gates, attention masking, and many other places.
Rank of a Matrix
Number of linearly independent rows/columns. Low-rank matrices are common in efficient AI (LoRA, adapters, distillation).

Question 1 of 10
In a neural network layer with input dimension 512 and output dimension 256, what is the shape of the weight matrix W?
✅ Correct! The weight matrix maps from input_dim → output_dim, so shape is (input_dim, output_dim) = (512, 256). The operation is X @ W where X is (batch, 512) → output is (batch, 256).
❌ Not quite. For a layer transforming 512 inputs to 256 outputs: W must have shape (512, 256) so that X @ W works: (batch, 512) · (512, 256) = (batch, 256). The inner dimensions must match!
Question 2 of 10
You compute the cosine similarity between two word embeddings and get a value of 0.98. What does this tell you?
✅ Correct! Cosine similarity of 0.98 (close to 1.0) means the vectors point in nearly the same direction → the words are semantically very similar. Example: "car" and "automobile" would have high cosine similarity.
❌ Incorrect. Cosine similarity ranges from -1 to 1. A value of 0.98 (close to +1) means the vectors are nearly parallel → semantically very similar. Note: cosine similarity is magnitude-independent (it only measures direction).
Question 3 of 10
Which operation is at the core of the Transformer's self-attention mechanism?
✅ Correct! Attention(Q,K,V) = softmax(Q·Kᵀ / √d_k)·V — the core is Q·Kᵀ, a dot product between query and key matrices. High dot product → high attention weight → "attend to this token".
❌ Incorrect. The self-attention mechanism computes dot products between query (Q) and key (K) vectors: softmax(Q·Kᵀ / √d_k). These dot products measure how relevant each position is to every other position — the attention scores.
Question 4 of 10
In NumPy, what does the expression A @ B compute?
✅ Correct! In Python 3.5+, the @ operator is matrix multiplication (matmul). This is equivalent to np.matmul(A, B). Element-wise multiplication uses A * B. You'll use @ constantly when working with PyTorch and NumPy.
❌ Incorrect. The @ operator in NumPy (and PyTorch) performs matrix multiplication. Element-wise multiplication uses the * operator. This distinction is critical — confusing the two is a very common bug in neural network code!
Question 5 of 10
Principal Component Analysis (PCA) is based on finding the eigenvectors of which matrix?
✅ Correct! PCA computes the covariance matrix Σ = (1/n) XᵀX, then finds its eigenvectors. These eigenvectors (principal components) point in the directions of maximum variance. Eigenvectors with the largest eigenvalues capture the most information.
❌ Incorrect. PCA finds eigenvectors of the data's covariance matrix. The covariance matrix encodes how features vary together. Eigenvectors of this matrix are the principal components — directions along which the data varies most.
Question 6 of 10
LoRA (Low-Rank Adaptation) fine-tuning represents weight updates as ΔW = A · B. What mathematical concept does this leverage?
✅ Correct! LoRA uses the insight that weight updates for fine-tuning have low intrinsic rank. Instead of updating W (e.g., 4096×4096 = 16M params), it learns small matrices A (4096×r) and B (r×4096) where r≪4096. With r=8, that's just 65,536 parameters — a 99.6% reduction!
❌ Incorrect. LoRA exploits low-rank approximation. SVD tells us any matrix can be written as U·Σ·Vᵀ, and if we keep only the top-k singular values, we get the best rank-k approximation. LoRA applies this idea: the change in weights during fine-tuning is low-rank, so A·B approximates it with far fewer parameters.
Question 7 of 10
You have a batch of 64 samples, each with 128 features. You apply a linear layer with 64 output neurons. What is the shape of the output?
✅ Correct! Input X has shape (64, 128). Weight matrix W has shape (128, 64). Matrix multiply: (64, 128) · (128, 64) = (64, 64). The batch dimension (64 samples) is preserved; the feature dimension changes from 128 → 64.
❌ Incorrect. Follow the shapes: Input = (batch=64, features=128). W = (128, 64). Output = X @ W = (64, 128) · (128, 64) = (64, 64). The batch size stays 64, and features transform from 128 → 64.
Question 8 of 10
What does it mean for the determinant of a weight matrix to be 0?
✅ Correct! Det(A) = 0 means the matrix is singular (non-invertible). It collapses the input space — multiple different inputs produce the same output. In neural networks, this means the layer is losing information. This relates to "dead neurons" and why weight initialization matters.
❌ Incorrect. Determinant = 0 means the transformation collapses volume to zero — the matrix is singular/non-invertible. In practical terms, it means the transformation is rank-deficient: multiple input vectors map to the same output, losing information. This is generally bad in neural networks.
Question 9 of 10
In backpropagation, gradients flow backward through a layer using the transpose of the weight matrix (Wᵀ). Why?
✅ Correct! For y = Wx, the gradient ∂L/∂x = Wᵀ · (∂L/∂y). This follows directly from the chain rule applied to matrix operations. If W transforms (input_dim → output_dim), then Wᵀ transforms back (output_dim → input_dim), matching gradient dimensions correctly.
❌ Incorrect. The chain rule of calculus, applied to matrix operations, gives: for y = Wx, the gradient with respect to x is Wᵀ·(∂L/∂y). This is a mathematical consequence of how matrix derivatives work — the transpose is needed to match dimensions and correctly propagate the gradient through the linear transformation.
Question 10 of 10
Consider this NumPy code. What does it compute?
a = np.array([3, 4])
b = np.array([4, 3])
result = np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
✅ Correct! This is cosine similarity. dot(a,b) = 3·4 + 4·3 = 24. ‖a‖ = ‖b‖ = 5. Cosine sim = 24/25 = 0.96. Even though a and b have swapped values, they're highly similar vectors pointing in nearly the same direction — just rotated slightly.
❌ Incorrect. This formula is cosine similarity: dot(a,b) / (‖a‖ · ‖b‖). Let's calculate: dot([3,4],[4,3]) = 12+12 = 24. ‖[3,4]‖ = 5, ‖[4,3]‖ = 5. Result = 24/25 = 0.96. Very similar vectors! Euclidean distance would be ‖a-b‖ = ‖[-1,1]‖ = √2 ≈ 1.41.
