Professor's Opening

Every output from a neural network is a probability. The softmax layer outputs a probability distribution. The loss function measures how surprised the model is by the correct answer. Model confidence is a probability. You cannot deeply understand AI without understanding probability. This tutorial builds that foundation from first principles.

Section 1

Probability Basics & Rules

Probability is a number between 0 and 1 that quantifies uncertainty. In AI, we deal with probability constantly — the probability a token is the next word, the probability an image contains a cat, the probability a user will click an ad.

Fundamental Rules

Core Probability Rules
P(A) ∈ [0, 1] ← all probabilities between 0 and 1
P(Ω) = 1 ← something must happen

Sum rule: P(A) = Σ P(A, B) over all B
Product rule: P(A, B) = P(A|B) · P(B)
Complement: P(not A) = 1 - P(A)
Independence: P(A,B) = P(A)·P(B) if A⊥B

Conditional Probability

P(A|B) is the probability of A given that B is true. This is the most important concept in probabilistic AI — everything is conditioned on something.

Conditional Probability
P(A|B) = P(A, B) / P(B)

// Example: P(spam | contains "FREE!") = ?
// = P(contains "FREE!" AND spam) / P(contains "FREE!")
🤖 AI Connection — Language Models

A language model is fundamentally a conditional probability machine: P(next_word | all_previous_words). GPT generates text by repeatedly computing P(token_i | token_1, token_2, ..., token_{i-1}) and sampling from this distribution. Every sentence ever generated by an LLM is produced by chaining conditional probabilities.

Section 2

Probability Distributions

A probability distribution describes how probability is spread over possible outcomes. Understanding which distribution fits your data is one of the key skills of an AI engineer.

The Normal (Gaussian) Distribution

The most important distribution in ML. Weight initialization, noise modeling, and many natural phenomena follow a Gaussian distribution.

Gaussian Distribution
N(x; μ, σ²) = (1/√(2πσ²)) · exp(-(x-μ)² / (2σ²))

· μ (mu) = mean (center of the bell curve)
· σ (sigma) = standard deviation (spread)
· σ² = variance

// 68% of data within 1σ, 95% within 2σ, 99.7% within 3σ
🤖 AI Connection — Weight Initialization

When you initialize neural network weights, you typically sample from N(0, 0.01) — a Gaussian with mean 0 and tiny variance. If weights are too large (large σ), gradients explode. If too small (σ≈0), gradients vanish. The famous Xavier/Glorot initialization and He initialization are carefully designed Gaussian initializations that keep activations in a healthy range throughout the network.

Bernoulli & Binomial Distributions

Binary outcomes (0 or 1, yes or no, spam or not spam) follow a Bernoulli distribution. Binary classification models output Bernoulli probabilities.

Bernoulli Distribution
P(X=1) = p, P(X=0) = 1-p

// A sigmoid output in a binary classifier is p in this distribution
sigmoid(z) = 1/(1+e^(-z)) → outputs P(Y=1|X)

Categorical & Softmax

When there are K classes (K>2), we use a Categorical distribution. The softmax function converts raw model outputs (logits) into a valid categorical probability distribution.

Softmax — Converting Logits to Probabilities
softmax(zᵢ) = e^(zᵢ) / Σⱼ e^(zⱼ)

Properties: all outputs > 0, all outputs sum to 1 → valid probability distribution

// Example — 3-class classifier (cat, dog, bird):
logits z = [2.0, 1.0, 0.1]
softmax → [0.659, 0.242, 0.099] ← probabilities!
// Model says 65.9% confident it's a cat
Python
import numpy as np
import matplotlib.pyplot as plt

# Gaussian distribution
mu, sigma = 0, 1
x = np.linspace(-4, 4, 200)
pdf = (1 / (sigma * np.sqrt(2 * np.pi))) * np.exp(-0.5 * ((x - mu)/sigma)**2)

# Softmax function
def softmax(logits):
    exp_logits = np.exp(logits - np.max(logits))  # subtract max for numerical stability
    return exp_logits / exp_logits.sum()

logits = np.array([2.0, 1.0, 0.1])
probs = softmax(logits)
print(f"Logits: {logits}")
print(f"Probabilities: {probs.round(3)}")  # [0.659, 0.242, 0.099]
print(f"Sum: {probs.sum():.4f}")           # exactly 1.0

# Sigmoid for binary classification
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

z = np.array([-3, -1, 0, 1, 3])
print(f"Sigmoid outputs: {sigmoid(z).round(3)}")  # [0.047, 0.269, 0.5, 0.731, 0.953]
Section 3

Bayes' Theorem

Bayes' theorem is one of the most important equations in all of AI. It tells us how to update our beliefs when we receive new evidence.

Bayes' Theorem
P(θ|D) = P(D|θ) · P(θ) / P(D)

· P(θ|D) = Posterior: what we believe AFTER seeing data D
· P(D|θ) = Likelihood: how probable is data D given parameters θ
· P(θ) = Prior: what we believed BEFORE seeing data
· P(D) = Evidence: normalizing constant (often ignored)

// Posterior ∝ Likelihood × Prior

Intuitive Example: Medical Diagnosis

A disease affects 1% of the population. A test is 95% accurate. You test positive. What's the probability you actually have the disease?

Python — Bayes' Theorem
# P(disease) = 0.01 (prior — 1% of population)
# P(positive | disease) = 0.95 (sensitivity)
# P(positive | no disease) = 0.05 (false positive rate)

p_disease = 0.01
p_pos_given_disease = 0.95
p_pos_given_no_disease = 0.05
p_no_disease = 1 - p_disease

# Total probability of testing positive
p_positive = (p_pos_given_disease * p_disease +
              p_pos_given_no_disease * p_no_disease)

# Bayes' theorem: P(disease | positive)
p_disease_given_positive = (p_pos_given_disease * p_disease) / p_positive

print(f"P(disease | positive test) = {p_disease_given_positive:.1%}")
# Output: ~16.1% — much lower than you'd intuitively think!
# This is why low base rates matter enormously in ML model evaluation
🤖 AI Connection — Bayesian Deep Learning & Regularization

L2 regularization (weight decay) has a Bayesian interpretation: it corresponds to placing a Gaussian prior P(θ) on the weights, assuming they should be small. Dropout has a Bayesian interpretation as approximate variational inference. The prior P(θ) encodes our belief that simpler models (smaller weights) are more likely — this is why regularization prevents overfitting.

Section 4

Expectation, Variance & Covariance

Key Statistical Measures
Expected Value: E[X] = Σ xᵢ · P(xᵢ) ← weighted average
Variance: Var(X) = E[(X - E[X])²] = E[X²] - (E[X])²
Std Deviation: σ = √Var(X)
Covariance: Cov(X,Y) = E[(X-μₓ)(Y-μᵧ)]
Correlation: ρ(X,Y) = Cov(X,Y) / (σₓ · σᵧ) ∈ [-1, 1]
🤖 AI Connection — Batch Normalization

Batch Normalization (a critical technique in modern neural networks) works by computing the mean and variance of each feature across the batch, then normalizing: x̂ = (x - μ) / σ. This keeps the distribution of activations stable during training, making it much faster to train deep networks. Without it, the distribution shifts constantly — a problem called "internal covariate shift".

Section 5

Maximum Likelihood Estimation (MLE)

MLE is the principle behind training almost every neural network: find the parameters θ that make the observed training data most probable.

Maximum Likelihood Estimation
θ_MLE = argmax_θ P(D|θ) = argmax_θ Π P(xᵢ|θ)

// Products are numerically unstable → take log (log-likelihood)
θ_MLE = argmax_θ Σ log P(xᵢ|θ)

// Maximizing log-likelihood = minimizing negative log-likelihood (NLL)
θ_MLE = argmin_θ -Σ log P(xᵢ|θ) ← this IS your loss function!
💡

MLE = Loss Function Training

When you train a classification model with cross-entropy loss, you are doing MLE. The cross-entropy loss is the negative log-likelihood under a categorical distribution. When you train with MSE loss, you are doing MLE under a Gaussian noise model. Loss functions are probabilistic statements.

Cross-Entropy Loss = Negative Log-Likelihood
L = -Σᵢ y_true[i] · log(y_pred[i])

// For binary classification (BCE):
L = -(y·log(ŷ) + (1-y)·log(1-ŷ))

// If model says P("cat")=0.9 and true label IS cat:
L = -log(0.9) = 0.105 ← small loss (model was confident & right)

// If model says P("cat")=0.1 and true label IS cat:
L = -log(0.1) = 2.303 ← large loss (model was confident & wrong)
Section 6

Information Theory: Entropy & KL Divergence

Information theory gives us the language to talk about uncertainty, surprise, and the difference between distributions — all essential for understanding ML loss functions and generative models.

Entropy & KL Divergence
Shannon Entropy: H(P) = -Σ P(x) · log P(x)
← measures average surprise / uncertainty in distribution P

Cross-Entropy: H(P, Q) = -Σ P(x) · log Q(x)
← how surprised are we if we use Q to encode events from P?

KL Divergence: KL(P||Q) = Σ P(x) · log(P(x)/Q(x)) ≥ 0
← how different is Q from the true distribution P?

H(P, Q) = H(P) + KL(P||Q)
🤖 AI Connection — VAEs & Diffusion Models

Variational Autoencoders (VAEs) use KL divergence in their loss function: the VAE loss = reconstruction loss + KL(q(z|x) || p(z)). The KL term pushes the learned distribution q(z|x) to be close to a standard Gaussian prior p(z). Diffusion models (Stable Diffusion, DALL-E 3) are also trained using a form of KL divergence minimization. Understanding KL divergence unlocks the theory behind the most powerful generative AI systems.

⭐

Summary: Statistics as the Language of AI

Neural network outputs are probability distributions. Loss functions are likelihood principles. Training is Bayesian inference (implicitly). Normalization techniques use variance. Generative models minimize distributional distances. Every single concept in this tutorial directly maps to a component of real AI systems. You'll see these connections daily as an AI engineer.

P-value
Used in A/B testing ML models. If p < 0.05, the difference between model A and B is statistically significant (not due to random chance).
Central Limit Theorem
The average of many independent random variables tends toward a Gaussian. Why mini-batch gradient estimates are reliable even with random sampling.
MAP Estimation
Maximum A Posteriori — like MLE but includes the prior. MAP with Gaussian prior = L2 regularization. MAP with Laplace prior = L1 regularization.
Mutual Information
How much knowing X tells us about Y. I(X;Y) = KL(P(X,Y) || P(X)P(Y)). Used in feature selection, information bottleneck theory.

Question 1 of 10
A language model generates text by repeatedly computing which probability?
✅ Correct! Language models are autoregressive — they compute P(tokenᵢ | token₁, ..., token_{i-1}) at each step. This conditional probability is what makes contextual understanding possible. The full sentence probability is the product of all these conditionals.
❌ Incorrect. Language models are autoregressive conditional probability machines. They compute P(next_token | all_previous_tokens) at each step. This is the conditional probability from Section 1 — and it's the fundamental operation that allows GPT and Claude to generate coherent text.
Question 2 of 10
The softmax function converts logits [3.0, 1.0, 0.0] to probabilities. What property do the output probabilities always satisfy?
✅ Correct! Softmax produces a valid probability distribution: all outputs > 0 (due to exp) and they sum to 1 (due to normalization). For logits [3,1,0], softmax gives approximately [0.844, 0.114, 0.042]. This is a valid categorical probability distribution.
❌ Incorrect. Softmax(z_i) = exp(z_i) / Σexp(z_j). Since exp() is always positive and we divide by the sum, all outputs are positive AND sum to 1 — making it a valid probability distribution. Equal values (1/K) would only happen if all logits are equal.
Question 3 of 10
In Bayes' theorem P(θ|D) ∝ P(D|θ) · P(θ), what does P(θ) represent in the context of neural network training?
✅ Correct! P(θ) is the prior — our belief about parameters before seeing training data. In practice, L2 regularization corresponds to a Gaussian prior: we believe weights should be near zero. This is MAP (Maximum A Posteriori) estimation, which extends MLE with a prior belief.
❌ Incorrect. P(θ) is the prior probability of the parameters — what we believe about the weights before seeing any training data. A Gaussian prior P(θ) ~ N(0, σ²) corresponds to L2 regularization in practice. It encodes the belief that weights should be small.
Question 4 of 10
Training a neural network with cross-entropy loss is equivalent to which statistical principle?
✅ Correct! Cross-entropy loss L = -Σ y·log(ŷ) is exactly the negative log-likelihood under a categorical distribution. Minimizing it = maximizing the likelihood P(D|θ) = Π P(yᵢ|xᵢ,θ). This is the probabilistic justification for why cross-entropy is the standard classification loss.
❌ Incorrect. Cross-entropy loss IS the negative log-likelihood: -log P(y|x,θ). Minimizing it = maximizing P(y|x,θ) = Maximum Likelihood Estimation. Understanding this connection is what separates engineers who know ML from those who truly understand it.
Question 5 of 10
Batch Normalization normalizes activations using which statistics computed over the batch?
✅ Correct! BatchNorm computes the mean μ and variance σ² across the batch for each feature, then normalizes: x̂ = (x - μ) / √(σ² + ε). This ensures activations stay centered near zero with unit variance, preventing the distribution from shifting erratically during training.
❌ Incorrect. Batch Normalization uses the mean μ and variance σ² (from Section 4). The normalization x̂ = (x - μ) / σ ensures zero mean and unit variance. It then applies learnable scale γ and shift β: y = γ·x̂ + β to allow the network to "un-normalize" if needed.
Question 6 of 10
A model predicts P(cat) = 0.9 and the correct label is indeed "cat". What is the cross-entropy loss for this single prediction?
✅ Correct! L = -log(0.9) ≈ 0.105. When the model is confident AND correct, the loss is small. For comparison: if the model said P(cat) = 0.1 when the true label is cat, loss = -log(0.1) = 2.303 — over 20× larger! This is why cross-entropy heavily penalizes confident wrong predictions.
❌ Incorrect. Cross-entropy loss = -y·log(ŷ) = -1·log(0.9) = -(-0.10536) ≈ 0.105. Since y=1 (correct class) and ŷ=0.9 (high confidence correct prediction), the loss is small. 2.303 would be the loss for ŷ=0.1 — when the model is confidently wrong.
Question 7 of 10
KL divergence KL(P||Q) has what key mathematical property?
✅ Correct! KL divergence is non-negative (by Gibbs' inequality) and equals 0 only when P = Q exactly. Importantly, KL(P||Q) ≠ KL(Q||P) — it's NOT symmetric. VAEs use this: they minimize KL(q(z|x) || p(z)) to force the learned encoder to be close to the prior, but the direction matters.
❌ Incorrect. KL(P||Q) ≥ 0 always (it's a non-negative measure of distributional difference), and = 0 iff P = Q. It is NOT symmetric: KL(P||Q) ≠ KL(Q||P) in general. KL can also be greater than 1, so it's not bounded at 1.
Question 8 of 10
Adding L2 regularization (weight decay) to a neural network's loss function has which Bayesian interpretation?
✅ Correct! L2 regularization adds λ·Σwᵢ² to the loss, which is equivalent to MAP with a Gaussian prior P(w) ~ N(0, 1/2λ). Note: L1 regularization corresponds to a Laplace prior (promotes sparsity). This Bayesian view explains why regularization reduces overfitting — it encodes a prior belief that weights should be small.
❌ Incorrect. L2 regularization (||w||²) corresponds to MAP with a Gaussian prior on weights. Pure MLE has no regularization. L1 regularization (||w||₁) corresponds to MAP with a Laplace prior (which promotes sparsity — this is why L1 is used for feature selection).
Question 9 of 10
The Central Limit Theorem is why mini-batch gradient descent works reliably. What does it tell us?
✅ Correct! The CLT says: the average of n independent samples tends toward N(μ, σ²/n) as n → ∞. This means a mini-batch gradient is an unbiased estimate of the full-batch gradient, and with enough samples it's reliable. This justifies using 32 or 64 samples instead of the whole dataset each step.
❌ Incorrect. The CLT tells us that the sample mean of n independent draws tends toward a Gaussian centered at the true mean. For gradient descent: averaging gradients over a mini-batch gives an unbiased estimate of the true gradient, and the CLT guarantees this estimate is reliable (Gaussian distributed around the true value).
Question 10 of 10
A disease has a 0.1% base rate. A perfect test (no false positives, no false negatives) comes back positive. What is P(disease | positive test)?
✅ Correct! If the test has 0% false positives (never positive when disease-free) and 0% false negatives (always positive when diseased), then a positive result means you definitively have the disease: P(disease|positive) = 1.0 = 100%. The base rate only matters when the test is imperfect. This is why Bayes' theorem with perfect tests reduces to pure logic.
❌ Incorrect. A perfect test has 0 false positives: P(positive | no disease) = 0. Applying Bayes: P(disease|+) = [P(+|disease)·P(disease)] / P(+) = [1.0 × 0.001] / [1.0×0.001 + 0×0.999] = 0.001/0.001 = 1.0 = 100%. With a perfect test, a positive is definitive. The base rate only matters when the test makes errors.
