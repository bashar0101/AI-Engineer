Section 1

Derivatives & Gradients

The derivative of a function measures how much the output changes when the input changes slightly. In AI, derivatives tell us: if I change this weight by a tiny amount, how does the loss change?

Derivative — Rate of Change
f'(x) = df/dx = lim[h→0] (f(x+h) - f(x)) / h

// Key rules you'll use constantly:
d/dx [xⁿ] = n·xⁿ⁻¹ ← power rule
d/dx [eˣ] = eˣ ← exponential
d/dx [ln(x)] = 1/x ← log (appears in cross-entropy)
d/dx [σ(x)] = σ(x)·(1-σ(x)) ← sigmoid derivative
d/dx [tanh(x)] = 1 - tanh²(x) ← tanh derivative

The gradient generalizes the derivative to multiple dimensions — it's a vector of all partial derivatives pointing in the direction of steepest ascent of the loss function.

Gradient — Multi-dimensional Derivative
∇f(x₁, x₂, ..., xₙ) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]

The gradient points toward the steepest increase.
The negative gradient points toward the steepest decrease.

// In neural networks: ∇_W L(W) points toward increasing loss
// We update W in the NEGATIVE gradient direction to decrease loss
🤖 AI Connection — Loss Landscape

The loss function of a neural network is a surface in a space with billions of dimensions (one per weight). Training is the process of navigating this landscape to find a valley (low loss region). The gradient tells us which direction is "uphill" — and we go downhill instead. This is exactly why it's called gradient descent.

Section 2

Partial Derivatives

A neural network has millions of parameters. A partial derivative ∂L/∂wᵢ measures how the loss changes when we nudge only weight wᵢ while holding everything else fixed.

Partial Derivatives
f(x, y) = x²y + 3xy²

∂f/∂x = 2xy + 3y² ← treat y as constant
∂f/∂y = x² + 6xy ← treat x as constant

// In a neural network with loss L and weights W:
∂L/∂W[i,j] = "how does loss change if I nudge weight W[i,j]?"
// Positive → increasing W[i,j] increases loss (bad)
// Negative → increasing W[i,j] decreases loss (good)
Python — Numerical vs Analytical Gradients
import numpy as np

# Numerical gradient (finite differences) — useful for debugging
def numerical_gradient(f, x, h=1e-5):
    grad = np.zeros_like(x)
    for i in range(x.size):
        x_plus  = x.copy(); x_plus.flat[i]  += h
        x_minus = x.copy(); x_minus.flat[i] -= h
        grad.flat[i] = (f(x_plus) - f(x_minus)) / (2 * h)
    return grad

# Example: L = sum(W^2) — simple quadratic loss
def loss(W): return np.sum(W**2)
def analytical_grad(W): return 2 * W  # d/dW [W^2] = 2W

W = np.array([[1.0, -2.0], [3.0, 0.5]])
num_grad  = numerical_gradient(loss, W)
anal_grad = analytical_grad(W)

print("Numerical gradient:\n", num_grad)
print("Analytical gradient:\n", anal_grad)
print("Match:", np.allclose(num_grad, anal_grad))  # True
Section 3

The Chain Rule — Heart of Backpropagation

The chain rule computes derivatives of composed functions. Since a neural network is just a composition of functions (layer₁ → layer₂ → ... → loss), the chain rule is backpropagation.

Chain Rule
If z = f(g(x)), then: dz/dx = (dz/dg) · (dg/dx)

// Multi-layer example — 3-layer network:
x → [W₁] → h₁ → [σ] → a₁ → [W₂] → h₂ → [loss] → L

∂L/∂W₁ = (∂L/∂h₂) · (∂h₂/∂a₁) · (∂a₁/∂h₁) · (∂h₁/∂W₁)

Backpropagation = efficiently applying chain rule to all weights
// Gradients flow backward through the network, layer by layer
💡

The Computational Graph

PyTorch's autograd system builds a computational graph as you do the forward pass, then traverses it backward to apply the chain rule automatically. When you call loss.backward(), PyTorch uses the chain rule across the entire graph. You never compute gradients by hand — the framework does it for you.

Python — PyTorch Autograd (Chain Rule in Action)
import torch

# Create tensors with gradient tracking
x  = torch.tensor([2.0], requires_grad=True)
W1 = torch.tensor([3.0], requires_grad=True)
W2 = torch.tensor([4.0], requires_grad=True)

# Forward pass — PyTorch builds the computation graph
h1 = W1 * x          # h1 = 3*2 = 6
h2 = W2 * h1         # h2 = 4*6 = 24
L  = h2 ** 2         # L  = 24^2 = 576

# Backward pass — chain rule applied automatically!
L.backward()

# Analytical: dL/dW1 = dL/dh2 * dh2/dh1 * dh1/dW1
#           = 2*h2   * W2      * x
#           = 2*24   * 4       * 2 = 384
print(f"dL/dW1 = {W1.grad.item()}")  # 384.0 ✓
print(f"dL/dW2 = {W2.grad.item()}")  # 288.0
print(f"dL/dx  = {x.grad.item()}")   # 576.0
Section 4

Gradient Descent — How Neural Networks Learn

Gradient descent is the optimization algorithm that uses gradients to iteratively update weights toward a lower loss.

Gradient Descent Update Rule
W ← W - α · ∇_W L(W)

where:
· W = current weights
· α = learning rate (step size)
· ∇_W L = gradient of loss w.r.t. weights

Variants:
Batch GD: use ALL training samples to compute gradient
Stochastic GD: use ONE sample per update (noisy but fast)
Mini-batch GD: use N samples (default in DL, e.g. batch_size=32)
Python — Gradient Descent from Scratch
import numpy as np
import matplotlib.pyplot as plt

# Simple example: minimize f(w) = w^2 (minimum at w=0)
def f(w):     return w**2
def grad_f(w): return 2 * w

w = 5.0   # starting point
lr = 0.1  # learning rate

print(f"Starting: w={w:.3f}, loss={f(w):.3f}")
for step in range(20):
    grad = grad_f(w)
    w = w - lr * grad           # gradient descent update
    if step % 5 == 0:
        print(f"Step {step:2d}: w={w:.4f}, loss={f(w):.6f}")

# Step  0: w=4.0000, loss=16.0
# Step  5: w=1.3107, loss=1.7180
# Step 10: w=0.4295, loss=0.1845
# Step 15: w=0.1408, loss=0.0198 → approaching 0!

# Effect of learning rate
for lr, name in [(0.01, "too small"), (0.1, "just right"), (1.1, "too large")]:
    w_test = 5.0
    for _ in range(10): w_test -= lr * 2 * w_test
    print(f"lr={lr} ({name}): final w={w_test:.3f}")
🤖 AI Connection — The Learning Rate is Critical

Too small a learning rate: training is painfully slow (thousands of epochs to converge). Too large: the loss oscillates or diverges (explodes). This is why learning rate scheduling (Section 6) is a core engineering technique. The learning rate is often the single most important hyperparameter in any training run.

Section 5

Modern Optimizers: SGD, Momentum, Adam

Vanilla gradient descent is rarely used in practice. Modern optimizers add adaptive learning rates and momentum to converge faster and more reliably.

SGD with Momentum
v ← β·v - α·∇L ← accumulate velocity
W ← W + v ← update weights

· β = momentum coefficient (typically 0.9)
· v = velocity vector (running average of gradients)

// Momentum: like a ball rolling downhill — builds up speed
// in consistent directions, dampens oscillations
Adam Optimizer (Adaptive Moment Estimation)
m ← β₁·m + (1-β₁)·g ← 1st moment (mean of gradients)
v ← β₂·v + (1-β₂)·g² ← 2nd moment (variance of gradients)
m̂ = m / (1-β₁ᵗ) ← bias correction
v̂ = v / (1-β₂ᵗ) ← bias correction
W ← W - α · m̂ / (√v̂ + ε) ← parameter update

Defaults: α=0.001, β₁=0.9, β₂=0.999, ε=1e-8

Adam = SGD + momentum + per-parameter adaptive learning rates
⭐

Which Optimizer to Use?

Adam is the default for most deep learning tasks — it's robust and requires minimal tuning. SGD with momentum can outperform Adam when carefully tuned and is common in computer vision (ResNets are often trained with SGD). AdamW (Adam + weight decay) is the standard for LLM training. Start with Adam unless you have a reason to do otherwise.

Python — PyTorch Optimizers
import torch
import torch.nn as nn
import torch.optim as optim

model = nn.Linear(512, 256)  # simple linear layer

# Different optimizers — swap out easily
optimizer_sgd  = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
optimizer_adam = optim.Adam(model.parameters(), lr=1e-3, betas=(0.9, 0.999))
optimizer_adamw= optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)

# Standard training step pattern
def training_step(model, optimizer, x, y_true, loss_fn):
    optimizer.zero_grad()      # 1. clear old gradients (ALWAYS do this!)
    y_pred = model(x)          # 2. forward pass
    loss   = loss_fn(y_pred, y_true)  # 3. compute loss
    loss.backward()            # 4. backward pass (chain rule)
    optimizer.step()           # 5. update weights
    return loss.item()
Section 6

Learning Rate Schedules

The learning rate should not stay constant throughout training. Schedules dynamically adjust the learning rate to achieve faster early convergence and finer tuning late in training.

Common Learning Rate Schedules
Step Decay: α(t) = α₀ · γ^(t/step_size)
Cosine Annealing: α(t) = α_min + ½(α_max-α_min)(1+cos(πt/T))
Warmup + Decay: α(t) = α_max·(t/t_warmup) for t < t_warmup
α(t) = cosine_decay(t) for t ≥ t_warmup

// Warmup is standard for Transformers/LLMs — start low,
// ramp up, then slowly decay. Used in GPT, BERT, LLaMA.
Python — Learning Rate Scheduling in PyTorch
import torch.optim as optim

optimizer = optim.AdamW(model.parameters(), lr=1e-3)

# Cosine annealing — decays lr smoothly
scheduler = optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=100, eta_min=1e-6
)

# Warmup + cosine decay (standard for LLM training)
def get_lr_with_warmup(step, warmup_steps, total_steps, max_lr, min_lr):
    if step < warmup_steps:
        return max_lr * step / warmup_steps  # linear warmup
    import math
    progress = (step - warmup_steps) / (total_steps - warmup_steps)
    return min_lr + (max_lr - min_lr) * 0.5 * (1 + math.cos(math.pi * progress))

# Training loop with scheduler
for epoch in range(100):
    loss = training_step(model, optimizer, X, y, loss_fn)
    scheduler.step()   # update learning rate after each epoch
    if epoch % 10 == 0:
        current_lr = optimizer.param_groups[0]['lr']
        print(f"Epoch {epoch}: loss={loss:.4f}, lr={current_lr:.6f}")
💡

Gradient Clipping — Preventing Explosions

In RNNs and Transformers, gradients can become enormous (explode), causing unstable training. Gradient clipping caps the gradient norm: torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0). This is called before optimizer.step() and is standard practice in LLM training.

0 of 10 answered
Question 1 of 10
In gradient descent, why do we subtract the gradient instead of adding it?
✅ Correct! The gradient ∇L points in the direction of steepest increase of the loss. Since we want to decrease the loss, we move in the opposite direction: W ← W - α·∇L. This is the fundamental intuition of gradient descent.
❌ The gradient points toward increasing loss (steepest ascent). We want to decrease the loss, so we move in the negative gradient direction. Hence W ← W - α·∇L (subtract). Adding the gradient would be gradient ascent — used in adversarial attacks, not training.
Question 2 of 10
What does optimizer.zero_grad() do, and why must it be called every training step?
✅ Correct! PyTorch accumulates gradients by default (useful for gradient accumulation tricks). Without zero_grad(), the gradient from step t is added to the gradient from step t+1, corrupting the update. Forgetting zero_grad() is one of the most common bugs when writing custom training loops.
❌ zero_grad() clears the .grad attribute of all parameters. PyTorch accumulates gradients across backward() calls (this is intentional for gradient accumulation). If you forget to call it, gradients from previous batches add up to the current gradient, giving incorrect updates. Classic beginner bug!
Question 3 of 10
The chain rule applied to a neural network is known as:
✅ Correct! Backpropagation is the algorithm that applies the chain rule efficiently to compute ∂L/∂W for every weight W in the network. It traverses the computational graph in reverse (backward), accumulating gradients layer by layer using the chain rule.
❌ Backpropagation is the chain rule applied to neural networks. It computes gradients by traversing the computational graph backward from the loss to the inputs, applying the chain rule at each operation. This is what makes neural network training computationally feasible.
Question 4 of 10
Adam optimizer maintains which two quantities for each parameter?
✅ Correct! Adam tracks m (exponential moving average of gradients = first moment) and v (exponential moving average of squared gradients = second moment). The second moment gives per-parameter adaptive scaling: parameters with consistently large gradients get smaller effective learning rates, preventing instability.
❌ Adam maintains: m = running mean of gradients (first moment, β₁=0.9) and v = running mean of squared gradients (second moment, β₂=0.999). The update is α·m̂/√v̂ — dividing by √v̂ adapts the effective learning rate per parameter based on its gradient history.
Question 5 of 10
What is the derivative of the sigmoid function σ(x) = 1/(1+e^(-x))?
✅ Correct! σ'(x) = σ(x)·(1-σ(x)). This elegant form means we can compute the gradient using only the forward pass output — σ(x) is already computed! This is why sigmoid is computationally efficient in backpropagation. Max value of σ'(x) is 0.25 (at x=0), which contributes to the vanishing gradient problem in deep networks.
❌ The sigmoid derivative is σ'(x) = σ(x)·(1-σ(x)). Notice: the maximum value is 0.25 (at x=0). This means gradients are multiplied by at most 0.25 at each sigmoid layer. In a 10-layer network, the gradient becomes 0.25^10 ≈ 0.000001 — this is the vanishing gradient problem! ReLU was invented to solve this.
Question 6 of 10
What happens during training if the learning rate is set too high (e.g., α = 10.0)?
✅ Correct! A too-large learning rate causes the optimizer to take huge steps that jump over the minimum, landing somewhere with even higher loss — then jumping again. The loss bounces around (oscillates) or grows without bound (diverges). In practice, you often see NaN values in the loss — a sure sign of exploding gradients or too large a learning rate.
❌ A too-large learning rate causes divergence or oscillation. Imagine a ball rolling in a bowl — with a huge step, it flies past the bottom and up the other side, then back, never settling. In practice: the loss goes up instead of down, or you see NaN. This is why learning rate warmup (starting small, growing gradually) is important for large models.
Question 7 of 10
What is the correct order of operations in a PyTorch training step?
✅ Correct! The canonical PyTorch training loop order: (1) zero_grad() — clear old gradients; (2) forward pass — compute predictions; (3) compute loss; (4) loss.backward() — compute gradients via chain rule; (5) optimizer.step() — update weights using gradients. Memorize this sequence!
❌ The correct order is: zero_grad() → forward pass → compute loss → loss.backward() → optimizer.step(). You must zero gradients first (prevent accumulation), compute predictions forward, get the loss, run backprop to get gradients, then update weights. This 5-step pattern appears in virtually every PyTorch training loop ever written.
Question 8 of 10
Warmup learning rate schedules (used in GPT, BERT, LLaMA training) start with a small learning rate and increase it. Why?
✅ Correct! At the very start of training, random weights produce large, chaotic gradients that don't yet reflect the true loss landscape. Taking large steps with these unreliable gradients can send the model to a very poor region. Small initial steps (warmup) let the model "find its footing" before taking larger, more informed steps.
❌ Warmup is about gradient reliability. Early training gradients are based on random weights — they point in unreliable directions. Large updates with unreliable gradients can permanently damage the model's initialization. Warming up prevents this by using small steps until gradients become more reliable as the model begins to learn the data distribution.
Question 9 of 10
Mini-batch gradient descent uses batches of 32–256 samples per update. What is the key advantage over full-batch gradient descent?
✅ Correct! Mini-batches give: (1) many more updates per epoch (1M samples / batch_size=32 = 31,250 updates); (2) efficient GPU utilization via parallel matrix ops; (3) the noise from sampling can help escape sharp local minima. Full-batch is precise but slow and memory-intensive. Single sample (SGD) is noisy and slow on GPU.
❌ Mini-batches are a tradeoff: less accurate than full-batch (noisier estimate of true gradient), but: (1) far more parameter updates per epoch; (2) excellent GPU parallelism; (3) the stochastic noise can act as implicit regularization. Full-batch gradients are more accurate but computationally expensive for large datasets that don't fit in memory.
Question 10 of 10
Gradient clipping (e.g., clip_grad_norm_(params, max_norm=1.0)) is applied before optimizer.step(). What problem does it solve?
✅ Correct! Exploding gradients occur when gradients grow exponentially through many layers (especially in RNNs and early Transformer training). Clipping rescales the gradient vector so its L2 norm never exceeds max_norm. This stabilizes training without distorting the gradient direction — just scaling down its magnitude when it's too large.
❌ Gradient clipping addresses exploding gradients. When gradients become very large, weight updates are enormous and destabilize training (you see the loss suddenly spike to NaN). clip_grad_norm_ rescales the gradient vector: if ‖g‖ > max_norm, then g ← g·(max_norm/‖g‖). Direction is preserved, magnitude is capped. Standard in RNN and Transformer training.
