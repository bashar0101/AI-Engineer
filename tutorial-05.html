Professor's Opening

Garbage in, garbage out. 80% of a real AI engineer's time is spent on data — loading, cleaning, transforming, and understanding it. A model trained on poorly prepared data will always underperform. Master data wrangling and your models will too.

Section 1

The Data Science Workflow

Every AI project follows a cycle: Collect → Clean → Explore → Feature Engineer → Model → Evaluate → Deploy. The first three steps are often the most time-consuming and highest-leverage.

🤖 AI Connection

The quality of training data determines the ceiling of model performance. GPT-4 was trained on carefully filtered, deduplicated data. Even the best architecture cannot compensate for noisy, imbalanced, or mislabeled data. Data curation is an AI engineering core skill.

Feature Engineering

Feature engineering is the art of transforming raw data into informative representations for the model. Even deep learning benefits from thoughtful features.

Python — Common Transformations
from sklearn.preprocessing import StandardScaler, MinMaxScaler, LabelEncoder

# Normalization (Z-score) — zero mean, unit variance
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

# Log transform — compresses skewed distributions
df['log_price'] = np.log1p(df['price'])  # log(1+x) handles zeros

# Binning — turn continuous into categorical
df['age_group'] = pd.cut(df['age'], bins=[0,18,35,60,100],
                         labels=['teen','young','adult','senior'])

# Date features
df['hour']    = pd.to_datetime(df['timestamp']).dt.hour
df['weekday'] = pd.to_datetime(df['timestamp']).dt.dayofweek
df['is_weekend'] = (df['weekday'] >= 5).astype(int)

Section 6

EDA Checklist

Shape & Types
df.shape, df.dtypes — how many rows, columns, and what data types.
Missing Values
df.isnull().sum() — where are the gaps and how many?
Distribution
df.describe(), histplots — are features skewed? Are there outliers?
Class Balance
df['label'].value_counts() — imbalanced classes need special handling.
Correlations
Correlation matrix — are features redundant? Is target correlated with features?
Duplicates
df.duplicated().sum() — duplicate rows inflate training data artificially.

0 of 10 answered
Question 1 of 10
Which Pandas method lets you aggregate data by a categorical column (e.g., total sales per region)?
✅ Correct! groupby() splits the DataFrame by category, agg() applies aggregation functions (sum, mean, count). This pattern is used constantly in EDA and feature engineering.
❌ df.groupby('region').agg(total=('revenue','sum')) is the pattern. sort_values sorts rows. pivot reshapes the table. apply applies a function row/column-wise without grouping.
Question 2 of 10
What is the primary purpose of StandardScaler (Z-score normalization)?
✅ Correct! StandardScaler computes (x - μ)/σ per feature, giving zero mean and unit variance. This prevents features with large scales (e.g., income in thousands) from dominating features with small scales (e.g., age). MinMaxScaler maps to [0,1].
❌ StandardScaler applies (x - μ)/σ → zero mean, unit variance. MinMaxScaler maps to [0,1]. StandardScaler doesn't remove outliers or change data type. Scale normalization is critical when features have very different ranges — otherwise gradient descent converges slowly.
Question 3 of 10
You have a column 'price' with highly right-skewed distribution (many low values, a few very high). What transformation helps?
✅ Correct! Log transform compresses the long tail of skewed distributions. np.log1p (log(1+x)) handles zeros. This makes the distribution more Gaussian, which helps many ML models. StandardScaler doesn't address skewness — it just shifts and scales.
❌ Log transform is the standard fix for right-skewed positive features. np.log1p handles zero values (log(0) is undefined). After log transform, the large outliers become much less extreme and the distribution approaches Gaussian — which most ML algorithms implicitly assume.
Question 4 of 10
What is one-hot encoding, and when is it needed?
✅ Correct! ML models need numerical inputs. 'cat'/'dog'/'bird' cannot be fed directly. One-hot encoding converts category k into a binary vector: cat=[1,0,0], dog=[0,1,0], bird=[0,0,1]. pd.get_dummies() or sklearn's OneHotEncoder do this automatically.
❌ One-hot encoding converts categories to binary vectors. 'red','green','blue' → [1,0,0],[0,1,0],[0,0,1]. Without this, models would treat category codes as ordered numbers (3 > 2 > 1), which is incorrect for nominal categories. Neural networks with embedding layers learn their own representations instead.
Question 5 of 10
When would you use forward-fill (ffill) for missing value imputation?
✅ Correct! Forward-fill propagates the last observed value forward in time. This makes sense for time series (sensor readings, stock prices, IoT data) where a gap often means the value hasn't changed — not that it's unknown. For random missing data, mean/median imputation is better.
❌ Forward-fill (ffill) is specifically for time series. If a temperature sensor misses readings for 2 hours, carrying forward the last known temperature is reasonable. For non-temporal random missingness, use mean/median imputation or a dedicated imputer.
Question 6 of 10
A training curve shows train loss decreasing but validation loss increasing after epoch 20. What does this indicate?
✅ Correct! Classic overfitting signature: train loss ↓ while val loss ↑. The model has memorized the training set (including its noise) and loses generalization ability. Fixes: early stopping, more data, regularization (L2, dropout), or a simpler model.
❌ Train loss down, val loss up = overfitting. The divergence point is when the model stops learning general patterns and starts memorizing training-specific details. Visualization of training curves is the #1 debugging tool for deep learning — always plot them.
Question 7 of 10
What does df.describe() return?
✅ Correct! describe() gives: count, mean, std, min, 25th/50th/75th percentile, max — per numeric column. This is your first stop in EDA: quickly understand the scale, central tendency, and spread of each feature. dtypes() gives types, head() gives first rows, corr() gives correlations.
❌ df.describe() returns summary statistics (mean, std, min, max, quartiles) for all numeric columns. Combined with df.isnull().sum(), df.dtypes, and df.shape, it forms the core EDA toolkit. Learn to read it quickly — you'll use it on every new dataset.
Question 8 of 10
You have highly correlated features (correlation = 0.97). What risk does this create for ML models?
✅ Correct! Highly correlated features (multicollinearity) create redundancy. For linear models, it makes coefficients unstable. For all models, it wastes capacity and can slow training. PCA is one solution. For tree-based models it matters less, but for neural networks, redundant features add noise. Drop one of the correlated features or apply dimensionality reduction.
❌ Highly correlated features contain redundant information. For linear models, multicollinearity makes coefficients numerically unstable (small data changes → large coefficient swings). For all models, it adds noise without information value. Correlation heatmaps help identify these pairs — then drop one or use PCA to combine them.
Question 9 of 10
What is the purpose of a train/validation/test split?
✅ Correct! The 3-way split: Train (fit weights), Validation (tune hyperparameters, choose architecture, early stopping), Test (final unbiased performance estimate — used ONCE at the very end). Using test set for decisions introduces data leakage and overly optimistic metrics.
❌ Train/val/test serves 3 purposes: (1) Train: fit model parameters; (2) Val: tune hyperparameters, compare models, early stopping — all decisions that affect the model use the val set; (3) Test: held-out final evaluation you look at only once. Looking at test set multiple times introduces selection bias.
Question 10 of 10
df.duplicated().sum() returns 500 in a dataset of 10,000 rows. What should you do?
✅ Correct! Duplicates inflate the training set — the model sees those examples more often than it should, creating implicit over-weighting. df.drop_duplicates() removes exact duplicate rows. 500 duplicates in 10,000 rows (5%) is significant and should be removed before training.
❌ Duplicates should be dropped. They make the model treat certain samples as if they appear multiple times in the true data distribution, introducing bias. df.drop_duplicates() is the fix. Always check for duplicates as part of your EDA checklist — they're surprisingly common in real-world datasets.
